# Bandit Level 21 → Level 22

## Objective

The password for the next level is stored in a file that is created automatically by a scheduled task.

The challenge hint suggests checking the system's cron configuration to understand what is being executed in the background.

## What Is Cron?

Cron is a Linux service used to run commands or scripts automatically at scheduled times.

System-wide cron jobs are often stored in:

```bash
/etc/cron.d/
```

Each file in this directory can define scheduled tasks for specific users.

## Exploring the Cron Jobs

I moved into the cron directory and listed its contents:

```bash
cd /etc/cron.d/
ls
```

The directory contained three relevant files:

```text
cronjob_bandit22
cronjob_bandit23
cronjob_bandit24
```

Since I was logged in as `bandit21`, I started by inspecting the cron job for `bandit22`:

```bash
cat cronjob_bandit22
```

The file contained:

```text
@reboot bandit22 /usr/bin/cronjob_bandit22.sh &> /dev/null
* * * * * bandit22 /usr/bin/cronjob_bandit22.sh &> /dev/null
```

## Understanding the Cron Job

The first line uses:

```text
@reboot
```

This means that the script is executed once every time the system starts.

The second line is a standard cron schedule:

```text
* * * * *
```

This means that the script runs every minute.

Both commands run as the `bandit22` user:

```text
bandit22 /usr/bin/cronjob_bandit22.sh
```

The final part:

```text
&> /dev/null
```

redirects both standard output and error output to `/dev/null`.

In simple terms, any normal output or error message generated by the script is discarded instead of being displayed or saved.

## Inspecting the Script

The cron job executes this script:

```bash
cat /usr/bin/cronjob_bandit22.sh
```

The script contained:

```bash
#!/bin/bash
chmod 644 /tmp/t7O6lds9S0RqQh9aMcz6ShpAoZKF7fgv
cat /etc/bandit_pass/bandit22 > /tmp/t7O6lds9S0RqQh9aMcz6ShpAoZKF7fgv
```

## Understanding the Script

The script performs two actions.

First, it changes the permissions of a temporary file:

```bash
chmod 644 /tmp/t7O6lds9S0RqQh9aMcz6ShpAoZKF7fgv
```

Permission mode `644` means:

```text
Owner: read and write
Group: read only
Others: read only
```

This makes the file readable by all users.

Next, the script copies the password for `bandit22` into that temporary file:

```bash
cat /etc/bandit_pass/bandit22 > /tmp/t7O6lds9S0RqQh9aMcz6ShpAoZKF7fgv
```

The `>` operator redirects the output of the `cat` command into the specified file.

Because the file is readable by everyone, I could simply read it:

```bash
cat /tmp/t7O6lds9S0RqQh9aMcz6ShpAoZKF7fgv
```

## Password Retrieved

```text
tRae0UfB9v0UzbCdn9cY0gQnds9GF58Q
```

## Key Takeaway

This level introduces cron jobs and scheduled task analysis.

Important concepts from this level:

* Cron can execute scripts automatically on a schedule.
* `/etc/cron.d/` contains system-wide cron job definitions.
* `@reboot` runs a command when the system starts.
* `* * * * *` runs a command every minute.
* `&> /dev/null` discards standard output and error output.
* Scripts running as another user may create files that expose useful information.
* File permissions can determine whether another user can read a file.

## Useful Commands

```bash
ls /etc/cron.d/
cat /etc/cron.d/cronjob_bandit22
cat /usr/bin/cronjob_bandit22.sh
cat /tmp/t7O6lds9S0RqQh9aMcz6ShpAoZKF7fgv
```

## Final Notes

This level shows why scheduled tasks should always be inspected during Linux enumeration.

Cron jobs can reveal scripts, file paths, permissions, commands executed by privileged users, and temporary files created automatically by the system.
